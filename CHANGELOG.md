# 版本发布说明

**发布日期**：2026-07-04
**版本性质**：初始发布，包含阶段一至阶段三全部功能

---

## 版本概述

「做饭这件小事」微信小程序首个完整版本。一个面向个人 + 朋友使用的菜谱点餐小程序：朋友浏览菜品并下单，主人查看订单、管理菜品，按菜谱做法做菜。不涉及支付与商业化。

---

## 新增功能点

### 🍽️ 菜品浏览（阶段一）

- **首页分类浏览**：左侧分类栏（荤菜 / 素菜 / 汤品 / 主食 / 饮品）+ 右侧菜品列表，独立滚动
- **顶部标题区**：个人头像 + 小程序名称 + 简介
- **菜品详情页**：展示菜品图片、名称、所需食材（含用量）、所需调味料（含用量）、制作过程（有序步骤）

### 🛒 点餐下单（阶段二）

- **加入菜单**：首页菜品卡片、详情页均可一键加入/移出菜单，状态实时同步
- **微信身份识别**：朋友填写昵称（必填）+ 头像（可选），openid 由云开发自动获取
- **提交订单**：所选菜品写入云数据库，自动记录下单人、时间
- **订单查看**：朋友查看自己的历史订单；主人查看所有人的订单
- **主人权限**：openid 白名单判定，主人视角显示「👑 主人」标识

### 📝 菜品管理（阶段三）

- **菜品云端存储**：菜品数据迁移至云数据库，无需改代码即可加菜
- **添加菜品页**（仅主人可进）：支持选图（自动传云存储）、填菜名、选分类、动态增删食材/调味料/步骤
- **批量导入**：`initDishes` 云函数支持 JSON 参数批量导入菜品，或导入内置 10 道示例菜
- **图片云存储**：菜品图上传至云存储，读取时自动转临时链接显示

## 使用前提

- 正式 AppID（测试号不支持云开发）
- 已开通微信云开发，云环境 ID 已配置在 `app.js`
- `orders`、`dishes` 集合已创建，权限设为「仅创建者可读写」
- 5 个云函数已部署到云端

---

## 已知限制（本次不做）

- 暂不支持编辑/删除已有菜品（需去云开发控制台手动操作）
- 暂不支持菜品搜索
- 暂不支持下单备注（如「少辣」「不吃香菜」）
- 暂不支持订单采购清单汇总
- 订单每道菜点/不点，不选份数

> 以上为有意排除项，详见 `需求文档.md` 第十节，待实际使用后按需迭代。
