## 二、阶段三必做配置（否则首页空白、无法加菜）

### 第 1 步：新建 dishes 数据库集合

1. 云开发控制台 → 数据库 → 新建集合，名称 **`dishes`**。
2. 数据权限设为「**仅创建者可读写**」（读写都走云函数）。

### 第 2 步：部署 3 个新云函数

右键 `cloudfunctions/` 下对应文件夹 →「**上传并部署：云端安装依赖**」：
- `getDishes`（读菜品）
- `saveDish`（增/改菜品，主人权限）
- `initDishes`（一次性导入示例菜）

### 第 3 步：导入示例菜品（让首页有数据）

部署 `initDishes` 后，导入方式二选一：

**方式 A（推荐，省事）**：云开发控制台 → 云函数 → 找到 `initDishes` → 点「**云端测试**」→ 直接点「运行测试」→ 返回 `{ok:true, msg:"导入完成，共 10 道"}` 即成功。然后去数据库 `dishes` 集合看，应有 10 条数据。

**方式 B**：先不导示例菜，直接在小程序「添加菜品」页一条条加你自己的菜（见第四节）。

> 导入示例菜后，`initDishes` 云函数可保留（以后想重置数据再用）或删除，不影响运行。

### 第 4 步：验证

1. 重新编译小程序，首页应显示 10 道示例菜（5 个分类）。
2. 点菜 → 进详情页，菜谱信息正常。
3. 进订单页（你已是主人）→ 身份栏右侧有「➕ 加菜」按钮 → 点进去能填表添加菜品 → 保存后回首页能看到新加的菜。

## 四、如何添加自己的菜品

1. 进小程序 → 「我的订单」页 → 身份栏点「➕ 加菜」。
2. 填写：图片（可选，自动传云存储）、菜名、分类、食材、调味料、步骤。
3. 点「保存」→ 自动回首页，新菜已显示。

> 不用改代码、不用重新发版，随时加。
